## IP Whitelist for Prov AI Cloud

Prov AI needs to access your Redshift database via its outbound IP address. If you have network access controls applied, please add the Prov AI service IP to your VPC allowlist.

<Tip>
  Scroll to the bottom of the data source connection page in Prov AI to find the specific IP address for your instance.
</Tip>

To add an Amazon Redshift connection, select the **Redshift** option in the "Connect a data source" section.

---

## Connect

### Authentication Methods

Prov AI supports two authentication methods for Redshift:

#### 1. Username and Password
Select this if you have direct database credentials for your Redshift cluster.

#### 2. AWS Credentials (IAM)
Select this if you prefer AWS IAM authentication. This requires your **AWS Access Key ID**, **Secret Access Key**, and the **Redshift Cluster Identifier**.

<Note>
  If you choose AWS credentials, ensure your IAM user has the `AmazonRedshiftReadOnlyAccess` policy (or equivalent permissions) granted.
</Note>

### Connection Details

- **Display Name**: The display name for this connection in Prov AI.
- **Host**: The endpoint URL of your Redshift cluster (e.g., `my-cluster.us-east-1.redshift.amazonaws.com`).
- **Port**: Standard Redshift port (default: `5439`).
- **Database**: The specific database name you want to connect to.
- **Username**: Your Redshift database username.
- **Password**: The password for your database user.

---

## Select Tables

All tables in your connected Redshift dataset will be listed. Select the tables you want to use in Prov AI. Each selected table will be created as a **Data Model**.

---

## Define Relationships

Define the relationships among selected tables. Prov AI will suggest relationships based on existing primary and foreign keys in your Redshift schema.

---

## Troubleshooting

### VPC and Security Groups
Ensure your Redshift cluster’s VPC and security groups allow inbound connections from the Prov AI service IP on port **5439**.
