<Warning>
  **Exclusive to Prov AI**: While Prov SQL uses standard SQL syntax, it is built specifically for the Prov AI platform. Copying Prov SQL directly into other databases may cause errors due to syntax specialization.
</Warning>

**Prov SQL** is a powerful and intuitive query language designed specifically for Prov AI. It provides a standardized syntax that allows you to query data from various sources (BigQuery, PostgreSQL, etc.) using a unified dialect.

---

## Key Concepts

### Result Prov SQL
Every answer in Prov AI includes the generated Prov SQL statement. Click the **View Full SQL** button in your results area to see the breakdown.

### Original SQL Translation
If you are connected directly to a data source, Prov AI allows you to view the **Original SQL** (e.g., BigQuery, MySQL, Snowflake). Click **Show Original SQL** to translate the Prov SQL into the native dialect of your database.

<Info>
  You can copy the **Original SQL** directly to your clipboard for use in your database's native console.
</Info>

---

## Why use Prov SQL?

- **Standardized Syntax**: Use one consistent language across all your connected data sources. No more switching between dialects.
- **Simplified Querying**: Intuitive syntax makes it easier for humans (and AI) to read and audit complex queries.
- **Seamless Integration**: Prov SQL is deeply integrated with Prov AI’s semantic layer, enabling advanced modeling and easier access to data.

<Tip>
  Prov SQL is the bridge between your natural language questions and your database's raw data.
</Tip>
