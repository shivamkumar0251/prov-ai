## How It Works

<Steps>
  <Step title="Select Source">
    Choose your database or file type from the connection list.
  </Step>
  <Step title="Provide Credentials">
    Enter Host, Port, Username, Password, or API Keys securely.
  </Step>
  <Step title="Validation">
    Prov AI validates the connection natively.
  </Step>
  <Step title="Metadata Sync">
    Tables and column schemas are fetched automatically.
  </Step>
  <Step title="Ready to Query">
    The data is instantly available in the Data Hub for querying!
  </Step>
</Steps>

## Security & Governance

<Warning>
**Security Best Practices:**
- **Never expose raw credentials** in logs or custom UI elements. Trust only the secure vault.
- **Limit access permissions:** Always provide Prov AI with a **read-only** database role. It only needs to run `SELECT` statements!
- **Avoid connecting directly to production databases** if they cannot handle analytical loads. Use read replicas primarily.
</Warning>

---

## Connection Modes

- **Live Connection (Recommended):** Queries are executed directly on your database in real time. Absolute freshness.
- **Cached Mode:** Data is periodically synced. Faster interface performance, but slight data delay.

---

## Troubleshooting

- **Connection failed?** → Double check credentials, firewalls, and IP whitelisting.
- **Missing tables?** → Ensure the provided database role has the correct `GRANT SELECT` permissions.
- **Slow queries?** → Optimize your database indices or switch to Cached Mode.
