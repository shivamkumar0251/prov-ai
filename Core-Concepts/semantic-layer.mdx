A **Semantic Layer** is a logical architecture that sits between your raw database and the AI query engine. It maps complex database tables and columns into unified, business-friendly terms.

<Warning>
  LLMs hallucinate SQL queries if they are only given raw database tables with ambiguous names. The Semantic Layer is the critical bridge that grounds the AI in reality, ensuring 100% accuracy.
</Warning>

---

## Why is it so important?

### Without a Semantic Layer
- **Inconsistent Metrics**: Marketing calculates "Revenue" differently than Finance.
- **Confusion**: The AI guesses which tables to join, often leading to slow queries or explosions.
- **Incorrect Insights**: Bad data mapping leads to bad business decisions.

### With Prov AI's Semantic Layer
- **Single Source of Truth**: "Revenue" is explicitly defined as \`Price × Quantity - Discount\`.
- **Consistent Results**: Everyone gets the same answer, no matter how they ask the question.
- **Perfect AI Understanding**: The LLM securely references your certified definitions, preventing hallucinations.

## What does it define?

The Semantic Layer standardizes three main components of your data stack:

<CardGroup cols={3}>
  <Card title="Metrics" icon="chart-line">
    The core calculations. (e.g., Gross Revenue, Active Users, Conversion Rate).
  </Card>
  <Card title="Dimensions" icon="cube">
    The attributes you slice metrics by. (e.g., Region, User Status, Time Hierarchies).
  </Card>
  <Card title="Relationships" icon="link">
    The join keys connecting fact tables and dimension tables together.
  </Card>
</CardGroup>

## Example Definition

Behind the scenes, the semantic layer explicitly teaches the AI how to compute derived fields so the user doesn't have to explain it every time:

```yaml
metric:
  name: Net Revenue
  description: Total revenue post-refunds and discounts
  sql: SUM(sales.amount - COALESCE(refunds.amount, 0))
```
